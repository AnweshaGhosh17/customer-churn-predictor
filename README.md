# ⚡ ChurnGuard — Customer Churn Predictor

> ML-powered churn prediction with SHAP explainability, CLV risk scoring, and actionable retention recommendations.

[![Deploy with Vercel](https://vercel.com/button)](https://vercel.com/new/clone?repository-url=https://github.com/YOUR_USERNAME/customer-churn-predictor)

## 🎯 What It Does

| Feature | Description |
|---|---|
| **Churn Predictor** | Enter customer details → get churn probability + risk label |
| **SHAP Explainability** | Visual breakdown of exactly WHY the model predicted churn |
| **What-If Simulator** | Change attributes and see probability shift in real time |
| **CLV × Risk Matrix** | Cross customer value with churn risk to prioritise retention |
| **Smart Recommendations** | Rule-based engine suggests the best retention action |
| **Risk Dashboard** | Overview of all customers with risk distribution charts |

## 🚀 Quick Start (Absolute Beginner Guide)

### Step 1 — Clone the repository

```bash
git clone https://github.com/YOUR_USERNAME/customer-churn-predictor.git
cd customer-churn-predictor
```

### Step 2 — Install Python dependencies

```bash
pip install -r requirements.txt
```

### Step 3 — Get the dataset

Download the Telco Customer Churn dataset from Kaggle:
https://www.kaggle.com/datasets/blastchar/telco-customer-churn

Save the file as: `data/raw/telco_churn.csv`

### Step 4 — Train the model

**Option A: Run the Python script**
```bash
python ml/train.py
```

**Option B: Use Google Colab** (recommended — no local setup needed)
1. Open `notebooks/03_model_training.ipynb` in Colab
2. Upload the dataset
3. Run all cells
4. The notebook will push the `.pkl` artifacts to GitHub automatically

### Step 5 — Start the API server

```bash
uvicorn api.index:app --reload
```

Open http://localhost:8000/docs to see and test all API endpoints.

### Step 6 — Open the frontend

Open `frontend/index.html` in your browser, OR use Live Server in VS Code.

> In `frontend/pages/predict.html`, the API URL is already set to `localhost:8000` when running locally.

## ☁️ Deploy to Vercel

### First time setup

```bash
npm install -g vercel    # install Vercel CLI
vercel login             # sign in with GitHub
```

### Deploy

```bash
vercel        # follow the prompts (say YES to defaults)
vercel --prod # promote to production
```

Vercel gives you a URL like `https://your-project.vercel.app`.

### Update the API URL in the frontend

After deploying, open these files and replace the API_BASE URL:
- `frontend/pages/predict.html`   — line with `API_BASE`
- `frontend/pages/simulator.html` — line with `API_BASE`
- `frontend/pages/dashboard.html` — line with `API_BASE`

Change `""` to `"https://your-project.vercel.app"` (your actual Vercel URL).

Commit and push — Vercel auto-redeploys on every push.

|---|---|---|
| `GET` | `/` | API info |
| `GET` | `/health` | Health check |
| `POST` | `/api/predict` | Predict churn for one customer |
